閱讀應用Library流程。

技術文件快速上手指南（jzx 教學邏輯）

🎯 目的
幫助工程師快速理解 .NET 類別庫或 API 文件，並正確應用指令與方法。

🧠 為什麼不先看命名空間？
命名空間（namespace）是模組分類，像是資料夾結構。
它不包含功能細節，也不告訴你怎麼用。
應該在最後確認引用是否正確，而不是當作入口。

✅ 正確閱讀順序

Step 1：先看「類別名稱」＝模組名稱
目的：了解這個模組是做什麼的
範例：ControllerVariableContainer 類別
類似你在工廠中辨識模組功能（例如：變數管理器、運動控制器）

Step 2：看「屬性」＝模組的輸入/設定點
目的：找出可以設定或取得的資料
範例：.Global、.Fieldbus、.Doubles
類似你設定模組參數（例如：速度、位置、變數空間）

Step 3：看「方法」＝模組的動作指令
目的：找出可以執行的功能
範例：GetMultiple 方法
類似你呼叫控制器指令（例如：MoveAbs、ArrayRead）

Step 4：看「範例」＝模組的使用流程
目的：直接學會怎麼用
範例：
double[] values = myController.Variables.Global.Doubles.GetMultiple(0, 200);
類似你參考教學流程圖或範例程式碼，快速套用

最後 Step：回頭確認命名空間
目的：確保你 using 了正確的命名空間
範例：
using Aerotech.A3200.Variables;
類似你確認模組路徑是否正確掛載

✅ 結語：這是一種「模組化閱讀策略」
你不是在「看文件」，你是在「拆模組」
你不是在「背 API」，你是在「組流程」
這就是 jzx 教學邏輯的精髓：結構化、可重用、可教學
