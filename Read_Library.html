閱讀應用Library流程。

🎯 目的
幫助工程師快速理解 .NET 類別庫或 API 文件，並正確應用指令與方法。

✅ 正確閱讀順序

Step 1：先看「類別名稱」＝模組名稱
目的：了解這個模組是做什麼的
範例：ControllerVariableContainer 類別
類似你在工廠中辨識模組功能（例如：變數管理器、運動控制器）

Step 2：看「屬性」＝模組的輸入/設定點
目的：找出可以設定或取得的資料
範例：.Global、.Fieldbus、.Doubles
類似你設定模組參數（例如：速度、位置、變數空間）

Step 3：看「方法」＝模組的動作指令
目的：找出可以執行的功能
範例：GetMultiple 方法
類似你呼叫控制器指令（例如：MoveAbs、ArrayRead）

Step 4：看「範例」＝模組的使用流程
目的：直接學會怎麼用
範例：
double[] values = myController.Variables.Global.Doubles.GetMultiple(0, 200);
類似你參考教學流程圖或範例程式碼，快速套用

最後 Step：回頭確認命名空間
目的：確保你 using 了正確的命名空間
範例：
using Aerotech.A3200.Variables;
類似你確認模組路徑是否正確掛載

✅ 結語：這是一種「模組化閱讀策略」
你不是在「看文件」，你是在「拆模組」
你不是在「背 API」，你是在「組流程」

🧩 把說明書當成「模組規格書」來看

元件 在說明書中叫什麼 你要看什麼
模組名稱 類別（Class） 它是做什麼的
模組參數 屬性（Property） 可以設定什麼
模組動作 方法（Method） 可以執行什麼
模組擴充 繼承（Inheritance） 是否有進階版本
模組分類 命名空間（Namespace） 屬於哪一類模組

✅ 給你一個實用技巧：只看你要用的那一段

  • 找到你要用的類別（例如 Variables）
  • 找到你要用的屬性（例如 Global.Doubles）
  • 找到你要用的方法（例如 GetMultiple()）
  • 看範例程式碼（直接複製來改）

📘 A3200 .NET Library 類別結構教學（jzx 教學邏輯）

Step 0：命名空間是分類資料夾
📄 Aerotech.A3200 命名空間
所有控制器相關類別都歸在此命名空間下，僅作為分類用途。

Step 1：Controller 是主入口類別
📄 Controller 類別
宣告：public class Controller : INamed<string>, IDisposable
實作代表類別具備特定功能合約，例如 IDisposable 要求提供 Dispose() 方法。

Step 2：Commands 是 Controller 的屬性
📄 Commands 屬性
宣告：public TaskSelectionCommands Commands { get; }
屬性型別是 TaskSelectionCommands，表示你取得的是任務選擇器物件，可進一步選擇任務並呼叫指令。

🔗 繼承：TaskSelectionCommands → RootCommands
📄 TaskSelectionCommands 類別
宣告：public class TaskSelectionCommands : RootCommands
提供 .Item(TaskId) 方法，讓你選擇任務（例如 T01）。

🔗 繼承：RootCommands → CommandCategory
📄 RootCommands 類別
📄 CommandCategory 類別
宣告：public class RootCommands : CommandCategory, INamed<TaskId>
提供 .Motion、.IO 等指令模組。

Step 3：Motion 是 RootCommands 的屬性
📄 Motion 屬性
宣告：public MotionCommands Motion { get; }
屬性型別是 MotionCommands，表示你取得的是一個運動指令模組物件。
因為 TaskSelectionCommands 繼承自 RootCommands，所以你可以透過 myController.Commands.Item(TaskId.T01) 直接存取 .Motion。

Step 4：MotionCommands 是運動指令集合
📄 MotionCommands 類別
宣告：public class MotionCommands : CommandCategory
所有運動指令都定義在這個類別中，例如 .Linear()、.Enable()。

Step 5：Linear() 是 MotionCommands 的方法
📄 Linear 方法
宣告：public void Linear(int[] axisIndexes, double[] Distance, double CoordinatedSpeed)
這是執行協調運動的指令，對應 AeroBasic 的 LINEAR 指令。

✅ 最終呼叫流程（jzx 教學流程）

// 1. 建立 Controller
var myController = Controller.Connect(...);

// 2. 選擇任務 T01
var task = myController.Commands.Item(TaskId.T01);

// 3. 呼叫運動指令
task.Motion.Linear(new[] { 0, 1 }, new[] { 10.0, 20.0 }, 5.0);

🧠 教學總結：類別關聯圖（文字版）

Controller
  └── Commands : TaskSelectionCommands
        └── (繼承) RootCommands
              └── (繼承) CommandCategory
              └── Motion : MotionCommands
                    └── (繼承) CommandCategory
