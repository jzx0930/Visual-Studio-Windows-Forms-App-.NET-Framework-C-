閱讀應用Library流程。

🧩 Step 0：閱讀目的與策略

🎯 目的：幫助工程師快速理解 .NET 類別庫或 API 文件，並正確應用指令與方法。

🔧 教學邏輯：你不是在「看文件」，你是在「拆模組」；不是在「背 API」，你是在「組流程」。

🧩 Step 1：先看「類別名稱」＝模組名稱

這是你理解模組功能的第一步。

範例：ControllerVariableContainer 類別

🔍 類似你在工廠中辨識模組功能（例如：變數管理器、運動控制器）

🔧 教學邏輯：先掌握模組的定位與用途。

🧩 Step 2：看「屬性」＝模組的輸入/設定點

這是你找出可以設定或取得的資料的地方。

範例：.Global、.Fieldbus、.Doubles

🔍 類似你設定模組參數（例如：速度、位置、變數空間）

🔧 教學邏輯：屬性是模組的介面，決定你能操作什麼。

🧩 Step 3：看「方法」＝模組的動作指令

這是你找出可以執行的功能的地方。

範例：GetMultiple() 方法

🔍 類似你呼叫控制器指令（例如：MoveAbs、ArrayRead）

🔧 教學邏輯：方法是模組的行為，代表它能做什麼。

🧩 Step 4：看「範例」＝模組的使用流程

這是你直接學會怎麼用的地方。

double[] values = myController.Variables.Global.Doubles.GetMultiple(0, 200);

🔍 類似你參考教學流程圖或範例程式碼，快速套用

🔧 教學邏輯：範例是模組的實戰應用，幫助你快速上手。

🧩 最後 Step：回頭確認命名空間

這是你確保引用正確的地方。

using Aerotech.A3200.Variables;

🔍 類似你確認模組路徑是否正確掛載

🔧 教學邏輯：命名空間是模組的分類結構，確保你能正確使用該類別。

✅ 教學總結：模組對照表

元件 在說明書中叫什麼 你要看什麼
模組名稱 類別（Class） 它是做什麼的
模組參數 屬性（Property） 可以設定什麼
模組動作 方法（Method） 可以執行什麼
模組擴充 繼承（Inheritance） 是否有進階版本
模組分類 命名空間（Namespace） 屬於哪一類模組

✅ 實用技巧：只看你要用的那一段

  • 找到你要用的類別（例如 Variables）
  • 找到你要用的屬性（例如 Global.Doubles）
  • 找到你要用的方法（例如 GetMultiple()）
  • 看範例程式碼（直接複製來改）

🧩 Step 0：命名空間是分類資料夾

📄 Aerotech.A3200 命名空間

所有控制器相關類別都歸在 Aerotech.A3200 命名空間下。

命名空間本身不包含功能，只是分類。

🔧 教學邏輯：這是「模組分類」，不看內容，只確認類別歸屬。

🧩 Step 1：Controller 是主入口類別

📄 Controller 類別

public class Controller : INamed<string>, IDisposable

是整個控制器操作的核心類別。

實作 INamed<string>（具名物件）與 IDisposable（可釋放資源）。

提供 .Commands、.Tasks、.Parameters 等屬性。

🔍 補充說明：
實作（Implements）指的是這個類別承諾會提供某些介面定義的方法或屬性。例如 IDisposable 要求類別提供 Dispose() 方法。
這是一種「功能合約」，讓類別具備特定能力。

🔧 教學邏輯：這是「模組入口」，所有功能都從這裡開始呼叫。

🧩 Step 2：Commands 是 Controller 的屬性

📄 Commands 屬性

public TaskSelectionCommands Commands { get; }

Commands 是 Controller 類別的屬性。

屬性型別是 TaskSelectionCommands，讓你選擇任務並取得該任務的指令模組。

🔍 補充說明：
屬性型別（Property Type）指的是這個屬性所對應的資料類型。在這裡，Commands 屬性的型別是 TaskSelectionCommands，表示你取得的是一個任務選擇器物件。
這決定了你可以對該屬性呼叫哪些方法或存取哪些子屬性。

🔧 教學邏輯：這是「模組參數」，你從這裡開始選擇任務。

🔗 繼承：TaskSelectionCommands → RootCommands

📄 TaskSelectionCommands 類別

public class TaskSelectionCommands : RootCommands

繼承自 RootCommands，提供 .Item(TaskId) 方法，讓你選擇任務（例如 T01）。

🔧 教學邏輯：這是「模組擴充」，你可以選擇任務並取得該任務的指令模組。

🔗 繼承：RootCommands → CommandCategory

📄 RootCommands 類別
📄 CommandCategory 類別

public class RootCommands : CommandCategory, INamed<TaskId>

繼承自 CommandCategory，實作 INamed<TaskId>，表示每個任務都有名稱。

提供 .Motion、.IO、.Program 等指令模組。

🔧 教學邏輯：這是「模組分類容器」，每個任務都可以取得一組指令模組。

🧩 Step 3：Motion 是 RootCommands 的屬性

📄 Motion 屬性

public MotionCommands Motion { get; }

屬性型別是 MotionCommands，提供運動控制指令，例如 .Enable()、.Linear() 等。

因為 TaskSelectionCommands 繼承自 RootCommands，所以你可以透過 myController.Commands.Item(TaskId.T01) 直接存取 .Motion。

🔧 教學邏輯：這是「模組動作入口」，你從這裡開始呼叫運動指令。

🧩 Step 4：MotionCommands 是運動指令集合

📄 MotionCommands 類別

public class MotionCommands : CommandCategory

繼承自 CommandCategory，所有運動指令都定義在這裡。

🔧 教學邏輯：這是「模組動作集合」，你會在這裡找到 .Linear()、.Enable() 等方法。

🧩 Step 5：Linear() 是 MotionCommands 的方法

📄 Linear 方法

public void Linear(int[] axisIndexes, double[] Distance, double CoordinatedSpeed)

執行協調運動，對應 AeroBasic 的 LINEAR 指令。

🔧 教學邏輯：這是「模組動作指令」，你實際執行控制器運動的地方。

✅ 最終呼叫流程（jzx 教學流程）

// 1. 建立 Controller
var myController = Controller.Connect(...);

// 2. 選擇任務 T01
var task = myController.Commands.Item(TaskId.T01);

// 3. 呼叫運動指令
task.Motion.Linear(new[] { 0, 1 }, new[] { 10.0, 20.0 }, 5.0);
