VisualStudio-WindowsFormsApp(.NET-Framework) 🎉

閱讀應用Library流程。

🎯 目的
幫助工程師快速理解 .NET 類別庫或 API 文件，並正確應用指令與方法。

✅ 正確閱讀順序

Step 1：先看「類別名稱」＝模組名稱
目的：了解這個模組是做什麼的
範例：ControllerVariableContainer 類別
類似你在工廠中辨識模組功能（例如：變數管理器、運動控制器）

Step 2：看「屬性」＝模組的輸入/設定點
目的：找出可以設定或取得的資料
範例：.Global、.Fieldbus、.Doubles
類似你設定模組參數（例如：速度、位置、變數空間）

Step 3：看「方法」＝模組的動作指令
目的：找出可以執行的功能
範例：GetMultiple 方法
類似你呼叫控制器指令（例如：MoveAbs、ArrayRead）

Step 4：看「範例」＝模組的使用流程
目的：直接學會怎麼用
範例：
double[] values = myController.Variables.Global.Doubles.GetMultiple(0, 200);
類似你參考教學流程圖或範例程式碼，快速套用

最後 Step：回頭確認命名空間
目的：確保你 using 了正確的命名空間
範例：
using Aerotech.A3200.Variables;
類似你確認模組路徑是否正確掛載

✅ 結語：這是一種「模組化閱讀策略」
你不是在「看文件」，你是在「拆模組」
你不是在「背 API」，你是在「組流程」

🧩 把說明書當成「模組規格書」來看

元件 在說明書中叫什麼 你要看什麼
模組名稱 類別（Class） 它是做什麼的
模組參數 屬性（Property） 可以設定什麼
模組動作 方法（Method） 可以執行什麼
模組擴充 繼承（Inheritance） 是否有進階版本
模組分類 命名空間（Namespace） 屬於哪一類模組

✅ 給你一個實用技巧：只看你要用的那一段

  • 找到你要用的類別（例如 Variables）
  • 找到你要用的屬性（例如 Global.Doubles）
  • 找到你要用的方法（例如 GetMultiple()）
  • 看範例程式碼（直接複製來改）
