VisualStudio-WindowsFormsApp(.NET-Framework) 🎉

小技巧筆記空間。

  1. 開始 F5
    啟動應用程式進行測試或執行。
  2. 建置解決方案 Ctrl+Shift+B
    快速編譯整個方案，檢查是否有語法錯誤。
  3. 加入中斷點 F9，跳出 Shift+F11
    用於偵錯流程控制，搭配 Visual Studio 的除錯工具。
  4. 摺疊制定義
    右鍵 → 大綱 → 摺疊制定義，可快速收起冗長程式區塊。
  5. 部署方式
    在 bin → Debug 資料夾中複製內容部署，避免被修改。（每台電腦配置不同）
  6. 尋找 Form 元件
    (1) Ctrl+Alt+T 開啟文件大綱快速定位元件。
    (2) 屬性頁面上方下拉選單可輸入元件首字母搜尋。
  7. 書籤功能
    Ctrl+K+K 插入書籤，Ctrl+K+W 開啟書籤視窗，Ctrl+K+N 快速跳回。
  8. 卷軸地圖模式
    工具 → 選項 → 文字編輯器 → C# → IntelliSense 可調整提示與說明。
  9. 工具與功能管理
    工具 → 取得工具與功能，可查看安裝時選擇的工具組，也能額外安裝舊版 .NET Framework。
  10. 查看 Framework 版本
    方案總管 → 專案右鍵 → 屬性，可查看該專案使用的 .NET Framework 版本。
  11. 輸出類型設定
    專案右鍵 → 屬性 → 應用程式 → 輸出類型。改為主控台應用程式可使用 Console.WriteLine()。

C# 方法回傳型別對照表

▶ 方法設計與回傳型別選擇

📌 定義與目的

在 C# 中，方法的回傳型別決定了呼叫端能取得什麼樣的結果。選擇合適的回傳型別能提升程式的可讀性、可維護性與擴充性。

📋 常見回傳型別與用途

回傳型別 用途說明 典型範例
void 不回傳任何資料，僅執行動作 UpdateUI(), LogError()
bool 回傳成功與否，流程控制 DoLaserProcessing(), TryConnect()
int / double 回傳數值結果 CalculateSpeed(), GetRetryCount()
string 回傳文字、訊息、路徑 GetErrorMessage(), GetFilePath()
object 回傳任意型別，通用容器 GetResult()
List<T> / T[] 回傳集合資料 GetAllFeatures(), FindMatchingFiles()
Dictionary<K,V> 回傳鍵值對，設定表 GetLaserSettings()
(T1, T2) 回傳多個值（Tuple） TryUploadFile() → (bool, string)
自訂類別 回傳完整結構與狀態 LaserProcessResult, JobReport
Task / Task<T> 非同步方法回傳型別 await SaveAsync(), await GetDataAsync()

🧠 設計建議

  • ✔ 想知道成功與否 → 使用 bool
  • ✔ 想要錯誤訊息 → 使用 string
  • ✔ 想要多個值 → 使用 Tuple 或 自訂類別
  • ✔ 想要非同步處理 → 使用 Task / Task<T>
  • ✔ 想要完整加工結果 → 使用 LaserProcessResult 類別

C# var 使用原則與最佳實踐

▶ var 是什麼？何時使用？

📌 定義與目的

var 是 C# 的編譯期型別推斷工具，讓程式更簡潔。編譯器會根據右側的值自動判斷變數型別。

📋 使用時機與建議

使用情境 說明 範例
型別明確 右側值清楚可辨 var count = 5; → int
型別冗長 簡化複雜型別宣告 var dict = new Dictionary<string, List<int>>();
LINQ 查詢 常用匿名型別 var result = from x in list select new { x, x * 2 };
foreach 迴圈 推斷集合元素型別 foreach (var item in list)

⚠️ 不建議使用的情況

  • ❌ 型別不明確 → 影響可讀性
  • ❌ 初學者維護 → 難以理解程式邏輯
  • ❌ 右側為 null → 編譯錯誤

🧠 實戰建議

  • ✔ 簡單型別可用可不用
  • ✔ 複雜型別建議使用 var
  • ✔ 團隊開發應遵守一致風格
  • ✔ var 是編譯期推斷，不是 dynamic
