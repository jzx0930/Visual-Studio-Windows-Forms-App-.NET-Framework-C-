VisualStudio-WindowsFormsApp(.NET-Framework) 🎉

小技巧筆記空間。

  1. 開始 F5
    啟動應用程式進行測試或執行。
  2. 建置解決方案 Ctrl+Shift+B
    快速編譯整個方案，檢查是否有語法錯誤。
  3. 加入中斷點 F9，跳出 Shift+F11
    用於偵錯流程控制，搭配 Visual Studio 的除錯工具。
  4. 摺疊制定義
    右鍵 → 大綱 → 摺疊制定義，可快速收起冗長程式區塊。
  5. 部署方式
    在專案內的主程式資料 bin → Debug 內的資料內容複製出來，儲存於另一新的資料夾，即可部署，如此一來程式無法被修改，也不會有被更改的風險。(但每台電腦的配置檔不同，需個別處理)
  6. 尋找 Form 元件
    (1) Ctrl+Alt+T 開啟文件大綱快速定位元件。
    (2) 屬性頁面上方下拉選單可輸入元件首字母搜尋。
  7. 書籤功能
    Ctrl+K+K 插入書籤，Ctrl+K+W 開啟書籤視窗，Ctrl+K+N 快速跳回。可在「檢視 → 書籤視窗」開啟。
  8. 卷軸地圖模式
    工具 → 選項 → 文字編輯器 → C# → IntelliSense 可調整提示與說明。卷軸可改為地圖模式。
  9. 工具與功能管理
    工具 → 取得工具與功能，可查看安裝時選擇的工具組，也能額外安裝舊版 .NET Framework。
  10. 查看 Framework 版本
    方案總管 → 專案右鍵 → 屬性，可查看該專案使用的 .NET Framework 版本。
  11. 輸出類型設定
    專案右鍵 → 屬性 → 應用程式 → 輸出類型。若改為主控台應用程式，可使用 Console.WriteLine() 並在 cmd 顯示結果。
  12. C# 方法回傳型別對照表
    依據方法的用途選擇適合的回傳型別，可提升程式可讀性與維護性。
    ✔ void：不回傳任何資料，適用於執行動作，如 UI 更新、寫入檔案。
    ✔ bool：回傳 true/false，適用於成功與否判斷，例如流程控制。
    ✔ int / double / float：回傳數值，適用於計算、統計、座標。
    ✔ string：回傳文字，適用於錯誤訊息、路徑、格式化輸出。
    ✔ object：回傳任意型別，適用於通用容器或動態資料。
    ✔ List / T[]：回傳集合，適用於查詢結果或批次資料。
    ✔ Dictionary：回傳鍵值對，適用於設定表或對應表。
    ✔ Tuple / ValueTuple：回傳多個值，適用於同時回傳狀態與資料。
    ✔ 自訂類別：回傳完整結構，適用於加工結果、錯誤資訊、統計報告。
    ✔ Task / Task：非同步方法回傳型別，搭配 async 使用。
  13. C# var 使用原則與最佳實踐
    var 是編譯期型別推斷，讓程式更簡潔，但需注意可讀性。
    ✔ 適合用在型別明確的情況，例如：var count = 5;
    ✔ 適合用在型別冗長的情況，例如：var dict = new Dictionary<string, List<int>>();
    ✔ LINQ 查詢與匿名型別幾乎必用 var。
    ⚠ 不建議用在型別不明確或右側為 null 的情況。
    ⚠ var 不是 dynamic，型別在編譯期就已確定。
    📌 實戰建議：簡單型別可用可不用，複雜型別建議使用，團隊開發應遵守一致風格。
