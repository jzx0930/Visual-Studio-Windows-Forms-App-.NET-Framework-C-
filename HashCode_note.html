HashCode 說明介紹

  • 快速比較
    HashCode 可用於快速比對字串或物件內容，效能遠高於逐字比較。
  • 唯一性標識
    雜湊碼通常能代表物件的內容，雖非絕對唯一，但碰撞機率極低。
  • 環境依賴性
    GetHashCode() 的結果可能因平台、架構或 .NET 版本而異，不適合跨平台儲存。

▼GetHashCode() 的用途與特性

用途說明

  • 快速比較：HashCode 能夠快速比較字串內容是否相同，尤其在檔案較大的情況下，比直接逐字比較更高效。
  • 唯一性標識：雜湊碼是一種特定的數學函式，其輸出通常能唯一表示該字串的內容（在絕大多數情況下）。

生成過程

  1. 字串內容轉換：將字串中的每個字符進行數值映射（通常基於 Unicode 編碼）。
  2. 數值運算：通過特定的算法（例如加權、組合）將所有字符的數值合併生成一個整數。
  3. 返回雜湊碼：最終生成一個整數，表示該字串的雜湊碼。

範例說明

以下程式碼會對字串 "Hello" 產生雜湊碼：


string example = "Hello";
int hashCode = example.GetHashCode();
Console.WriteLine(hashCode); // 例如：69639043（依環境而異）

※ 注意：雜湊碼的值是環境相關的，可能因平台或編譯器不同而有所差異。

特性總結

  • 快速：雜湊運算的效率通常比逐字比較高得多，尤其適合處理大量或長字串的情境。
  • 非唯一性：雜湊碼雖然通常唯一，但在極少數情況下，可能不同的字串會生成相同的雜湊碼（稱為雜湊碰撞）。
  • 環境依賴：GetHashCode() 的結果可能會因不同的 .NET 平台或版本而不同，並不適合跨平台使用。

▼雜湊碼的生成原理

運作步驟

  • 將字串中的每個字元轉換為 Unicode 數值。
  • 透過加權與組合算法，計算整體雜湊值。
  • 回傳整數型別的 HashCode。

範例程式碼


string example = "Hello";
int hashCode = example.GetHashCode();
Console.WriteLine(hashCode);

▼自訂類別的 HashCode 實作

覆寫範例


public class Person {
    public string Name;
    public int Age;

    public override bool Equals(object obj) {
        if (obj is Person other)
            return Name == other.Name && Age == other.Age;
        return false;
    }

    public override int GetHashCode() {
        return HashCode.Combine(Name, Age);
    }
}
