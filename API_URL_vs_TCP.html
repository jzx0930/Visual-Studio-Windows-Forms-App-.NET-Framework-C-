IP 和端口 vs. API URL

🔹 IP 和端口

  • IP 地址：例如：192.168.5.121，類似於網絡中的「家庭地址」，幫助其他設備找到這台機器。
  • 端口號：例如：5554，幫助確定要與該設備上的哪個應用程式或服務通信。每個服務使用不同的端口號。
  • 直接通信：通常是基於 TCP 或 UDP 的數據傳輸，不一定遵循特定的協議。

🔹 API URL 結構

例如：

http://192.168.5.121:5554/api/machines?status=active
  • 協議：http 或 https
  • 主機名：192.168.5.121（可以是 IP 地址或域名）
  • 端口：:5554
  • 路徑：/api/machines（指向特定資源）
  • 查詢參數：?status=active（提供額外的信息）

📡 基於 HTTP 的通信

用於獲取或操作數據，遵循一定的規範。

📌 總結

直接通信（使用 IP 和端口）是基於任意協議的數據傳輸。API URL 是一種標準化的方式，用於在網絡上進行數據請求和操作。
